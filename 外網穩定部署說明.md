# 外網穩定部署說明

目前 localhost.run 產生的網址屬於臨時外網穿透，會因連線中斷、電腦休眠、網路變動或服務回收而失效。若要提供外部單位穩定瀏覽，建議改用靜態網站正式部署。

## 已完成調整

- 已建立 `docs/` 乾淨發布資料夾。
- `docs/` 僅包含網頁必要檔案，不包含 PDF、報告、截圖或 tunnel log。
- 已新增 Render 設定：`render.yaml`。
- 已新增 Netlify 設定：`netlify.toml`。
- 已新增 GitHub Pages 支援檔：`docs/.nojekyll`。

## 推薦方案

### 方案 A：GitHub Pages

適合固定網址與長期分享。

1. 建立 GitHub repo。
2. 上傳本資料夾內容。
3. 到 repo 的 Settings > Pages。
4. Source 選 Deploy from a branch。
5. Branch 選 `main`，Folder 選 `/docs`。
6. 取得固定網址，例如：

```text
https://帳號.github.io/repo-name/
```

### 方案 B：Render Static Site

適合與既有 `onrender.com` 平台一致。

1. 登入 Render。
2. New > Static Site。
3. 連接 GitHub repo。
4. Publish Directory 設為：

```text
docs
```

5. 或直接使用根目錄的 `render.yaml`。本專案已設定：

```yaml
type: web
runtime: static
buildCommand: echo "Static site - no build step"
staticPublishPath: ./docs
```

6. 取得固定網址，例如：

```text
https://landslide-dam-risk-app.onrender.com/
```

### 方案 C：Netlify

適合快速拖曳部署。

1. 登入 Netlify。
2. 將 `docs/` 資料夾拖曳到 Netlify Drop。
3. 取得固定網址。

## 注意

正式部署版不需要本機電腦持續開機，也不需要 localhost.run tunnel。
